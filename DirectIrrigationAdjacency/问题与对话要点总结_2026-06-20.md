## 当前目标

定位并稳定《缺氧》本地模组 `DirectIrrigationAdjacency` 在最近版本中引发或关联的崩溃问题。当前优先级是先保证游戏可稳定进入和运行，再逐步恢复或验证额外功能。

## 已观察到的问题

1. 用户反馈：启用最近的环境液体吸取改动后，游戏再次崩溃。
2. 最新 `Player.log` 中可以看到：
   - `DirectIrrigationAdjacency.dll` 成功加载。
   - `PLib` 成功注册 `DirectIrrigationAdjacency.ModOptions`。
   - 出现 `[DirectIrrigationAdjacency] Loaded`。
   - `ThirdPartyHotFix` 的 `ElementConsumer.AddMass` 防护也成功加载。
3. 日志尾部没有看到 `DirectIrrigationAdjacency` 相关托管异常栈。
4. Windows 应用事件中出现 `OxygenNotIncluded.exe` 的 `APPCRASH`，故障模块为 `ntdll.dll`。
5. 初步判断：这次更像原生层硬崩或模拟层不稳定，而不是普通 C# 托管异常。

## 高风险变更

最近新增的 `ElementConsumerExtender.cs` 会在 `EntityTemplates.ExtendPlantToIrrigated` prefab 阶段为液体灌溉植物添加多个 `PassiveElementConsumer`：

- 4 个正交方向 consumer。
- 4 个斜角方向 consumer。
- 运行时再绑定到 `IrrigationMonitor.Instance` 的内部 `Storage`。

虽然该方案避免了更危险的运行时 `AddComponent` 和 `SimMessages.ModifyMass`，但它仍会显著增加植物 prefab 上的 sim consumer 数量。结合无托管栈的原生崩溃现象，它被列为当前最高嫌疑。

## 已完成的安全回滚

为优先稳定游戏，当前没有删除 `ElementConsumerExtender.cs`，而是新增总开关并默认关闭：

```json
{"AllowDiagonal":false,"EnableEnvironmentConsumers":false,"LiquidSupplyMultiplier":2.0,"SolidSupplyMultiplier":1.05}
```

关键调整：

1. `ModOptions.cs`
   - 新增 `EnableEnvironmentConsumers`。
   - 默认值为 `false`。
   - 描述标记为实验功能，当前默认关闭以避免原生层崩溃。
2. `ElementConsumerExtender.cs`
   - `ExtendPlantToIrrigated` postfix 入口先检查 `EnableEnvironmentConsumers`。
   - `WireRuntimeConsumers` 入口先检查 `EnableEnvironmentConsumers`。
   - `RefreshRuntimeConsumers` 入口先检查 `EnableEnvironmentConsumers`。
3. 默认配置下不会新增、绑定或刷新任何额外 `PassiveElementConsumer`。
4. 保留相邻 `Storage` 桥接逻辑：
   - 相邻种植箱/砖 Storage 液体桥接。
   - 固体肥料通过 `Storage.Transfer` 限速桥接。

## 当前复测建议

1. 保持当前配置：

```json
{"AllowDiagonal":false,"EnableEnvironmentConsumers":false,"LiquidSupplyMultiplier":2.0,"SolidSupplyMultiplier":1.05}
```

2. 进入相同存档复测。
3. 如果不再崩溃，基本可以确认新增环境 `PassiveElementConsumer` 扩展与崩溃强相关。
4. 如果仍然崩溃，下一步重点应转向：
   - `AdjacencySupply.cs` 中剩余的 `Storage.AddLiquid` 桥接。
   - 相邻 Storage 源对象生命周期是否可能失效。
   - 其他近期日志中反复出现异常的模组，如 FastTrack、翻译修复、TrueTiles 相关附属模组等。

## 后续处理路线

### 路线 A：当前安全版稳定

如果复测稳定：

1. 保持 `EnableEnvironmentConsumers=false` 作为默认发布配置。
2. 将环境液体吸取功能标记为实验功能。
3. 若仍要恢复环境液体功能，应另开测试分支，小范围限制：
   - 只给单一植物或单一元素添加 consumer。
   - 只启用 1 个 sample offset。
   - 不启用斜角。
   - 每一步都单独进游戏验证。

### 路线 B：当前安全版仍崩

如果复测仍崩：

1. 暂停环境 consumer 方向。
2. 在 `AdjacencySupply.cs` 中继续缩小范围。
3. 优先加开关禁用液体 `Storage.AddLiquid` 桥接，仅保留固体 `Storage.Transfer`。
4. 再根据新 `Player.log` 和 Windows 事件定位是本补丁还是其他模组。

## 本次对话结论

本次没有找到明确的 `DirectIrrigationAdjacency` 托管异常栈，但由于崩溃发生在新增环境 consumer 扩展后，且 Windows 事件显示原生层 `ntdll.dll` 崩溃，当前已采取最小安全回滚：默认完全关闭环境 `PassiveElementConsumer` 扩展，只保留相邻 `Storage` 桥接功能。下一步需要用户用当前安全版复测，以确认崩溃是否消失。
